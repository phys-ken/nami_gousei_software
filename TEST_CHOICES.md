# ブラウザテスト指示書 — Wave B 制限 + 選択肢モード（Type3/4）

**対象エージェント**: ブラウザを操作できる LLM Agent
**テスト URL**: http://localhost:8000
**前提**: `python server.py` が起動中であること
**目的**:
1. 波 B の有無による設問タイプ制限が正しく動作することを確認する（**①**）
2. Type3/Type4 の選択肢モードが正しく動作する（**②**）
3. 選択肢モードを使わない場合、**従来と同じ使用感**であることを確認する（既存退行確認）

---

## 重要な注意

- **既存挙動の保全が最優先**: 選択肢モードの「OFF」状態（デフォルト）で生成された問題・解答・PDF・ZIPは**従来と完全に同じ**でなければなりません。
- **テスト前に必要に応じて localStorage をクリア**: キーは `waveapp_choicesConfig`、`waveapp_cellSize`、`waveapp_styleMode`、`waveapp_styleConfig`、`waveapp_customStyleConfig`。
- 各テストでブラウザの開発者コンソール（F12）を開き、JS エラーが発生していないかも確認すること。
- アラート（`alert()` / `confirm()`）を含むテストでは、ダイアログのボタンを正しく押す。

---

## 事前準備

1. ブラウザで http://localhost:8000 を開く
2. ページが正常に表示されることを確認
3. 開発者コンソールを開いてエラーが出ていないか確認
4. 「設問作成」タブを開いておく

# 参考: アーキテクチャ

- 選択肢モードの状態は [App.choicesConfig](js/app.js) に保持
- `distractors[]` は `Wave` インスタンスの配列。長さ = `count - 1`（正答は別途自動生成）
- 正答 Canvas は [ProblemGenerator.renderType3CorrectCanvas](js/problems.js) / [renderType4CorrectCanvas](js/problems.js) で生成
- 不正解 Canvas は同じく `renderType3DistractorCanvas` / `renderType4DistractorCanvas`
- シャッフルは [Exporter.shuffleChoicesWithSeed](js/exporter.js) → 内部で [SeededRandom.seededShuffleIndices](js/random.js)
- シード = `hashString(問題波形JSON + パラメータ + 選択肢数)`
- PDF の 2列タイル描画は [Exporter._renderChoicesGridToPdf](js/exporter.js)

# 参考: 関連ユニットテスト

```bash
node --test tests/wave.test.js tests/renderer.test.js tests/random.test.js
```

合計 64 ケース。`SeededRandom.seededShuffle` の決定論性、ハッシュの安定性などはユニットテストで担保済み。
