## バグ一覧

### ✅ Bug 1: 問題 Canvas に合成波が描画されていた（修正済み）

**影響**: Type 4・Type 5

**症状**: 「t=○秒の合成波を描け」という設問の参照図（問題の Canvas）に、すでに合成波（実線）が描画されていた。

**修正内容**: `js/problems.js` に `_renderWavesOnly(waveA, waveB, t)` メソッドを追加し、`generateType4()`・`generateType5()` の `questionCanvases[0]` をこのメソッドに変更。波A・波Bの点線/破線のみ描画し合成波は含めない。

---

### ✅ Bug 2: 時刻ラベルと y 軸ラベルが重なっていた（修正済み）

**影響**: 全タイプ（特に xMin=0 のデフォルト設定時）

**症状**: グラフ左上の「y [cm]」（y軸ラベル）と「t = 0 [s]（初期波形）」（時刻ラベル）が重なって表示されていた。

**修正内容**: `js/renderer.js` の `drawTimeLabel()` の描画位置をグラフ右上に変更（`textAlign='right'`、x を `toPixel(xMax, 0).px`、y は上余白の 6px のまま）。

---

### ✅ Bug 3: Type 5 の問題文と参照図の時刻が不一致だった（修正済み）

**影響**: Type 5

**症状**: 問題文には「t = 0 のとき、下図のような…」と書かれているが、参照図の Canvas には `tStart` の時刻の波形が描かれていた。

**修正内容**: `generateType5()` の `questionCanvases[0]` を `_renderWavesOnly(waveA, waveB, 0)` に変更し、常に t=0 の初期状態を参照図として使用。`questionText` の記述（「t = 0 のとき」）と一致した。

---

### ✅ Bug 4: 入力値 0 が無効化されデフォルト値に置き換わっていた（修正済み）

**影響**: Type 1〜5 のすべてのパラメータ入力

**症状**: 各スピンボタンに `0` を入力して「設問を生成」しても、デフォルト値で生成されていた。

**修正内容**: `js/app.js` の `generateProblem()` 内で `|| デフォルト値` パターンを `isNaN()` チェックに変更。`_int(id, def)` / `_float(id, def)` ヘルパーを導入し、NaN のときのみデフォルト値を適用。0 が有効な入力として扱われるようになった。

---

### ✅ Bug 5: xMin > 0 のとき y 軸が画面外に消えていた（修正済み）

**影響**: グリッド設定で xMin を正の値に変更した場合

**症状**: `xMin=2` などに設定すると、y軸がグラフ描画エリア外に消えていた。

**修正内容**: `js/renderer.js` の `drawAxes()` で、`xMin >= 0` のとき `xAxis = Math.max(xLeft, toPixel(0,0).px)` としてグラフ左端にクリップ。xMin が負のとき（y 軸がグラフ内に入る場合）はそのまま x=0 の位置を使用。

---

### ✅ 追加: プレビュー Canvas が低解像度だった（修正済み）

**影響**: 進行波プレビュータブのプレビュー Canvas および PNG ダウンロード

**症状**: プレビュー Canvas は 580×200px（pixelRatio=1）で、設問生成 Canvas（1160×400px, pixelRatio=2）より低解像度だった。

**修正内容**: `js/app.js` の `renderPreview()` で Canvas を 1160×400px に変更し、`style.width/height` で 580×200px として表示。`pixelRatio: 2` を付与した WaveRenderer を使用。

## 🐛 新規バグ（追加テストで発見）

### Bug A: x 軸ラベルがキャンバス右端でクリップされる（未修正）

**影響**: 全タイプ・全 Canvas（エディタ・プレビュー・問題・解答すべて）

**症状**: x 軸ラベル「x [cm]」が「10 x [c」のように右端で切れ、「m]」が表示されない。  
Type 3 の y-t グラフでは「t [s]」の「]」が切れる。PNG ダウンロード・PDF にも同様に反映される。

**根本原因**: `js/renderer.js` `drawAxes()` のラベル描画位置が `xRight + 8` (論理ピクセル 560) で `textAlign='left'` のため、「x [cm]」(~35px) がキャンバス右端 (論理 580px) を超える。利用可能幅は 20px しかない。

```
logicalWidth=580, paddingRight=28
xRight = logicalWidth - paddingRight = 552  ← 常に一定
label開始 = xRight + 8 = 560
利用可能 = 580 - 560 = 20px  ←「x [cm]」(35px) が入らない
```

**修正案**: 以下のいずれかを推奨
- `paddingRight` を 28 → 52 に増やす（他のラベル位置も調整が必要な可能性あり）
- `textAlign = 'right'` に変えて位置を `xRight + 26` にする（矢印の先に右揃えで収める）
- フォントサイズを小さくする（10px serif に変更）

---

### Bug B: 速さ入力の `|| 1` パターンが未修正（低優先度）

**影響**: 波 A・波 B の速さ入力

**症状**: `js/app.js` の `_bindSpeedInputs()` および `generateProblem()` 内で  
`parseFloat(value) || 1` パターンが残存（Bug 4 と同じコードパターン）。

**注意**: `waveASpeed`・`waveBSpeed` 入力には `min="0.5"` が設定されているため、  
通常の UI 操作では speed=0 を入力できない。実際の影響はほぼなし。  
コードの一貫性として修正が望ましいが、優先度は低い。

**該当行**: `app.js` 142, 145, 222, 224 行  
`this.waveA.speed = parseFloat(e.target.value) || 1;`  
→ `const v = parseFloat(e.target.value); this.waveA.speed = isNaN(v) ? 1 : v;` に変更
